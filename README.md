# DEGtoTF

Скрипт на Python для автоматизации сопоставления данных о транскрипционных факторах (TF) с результатами дифференциальной экспрессии генов (DEG). 

Скрипт объединяет данные из трех таблиц, фильтрует их по уровню значимости (FDR) и формирует сводный отчет.

## Функционал

- **Интеграция данных**: Объединяет таблицу транскрипционных факторов с двумя таблицами результатов анализа (например, точки времени 24ч и 72ч).
- **Фильтрация по достоверности**: Переносит только те данные, где значение `FDR` (False Discovery Rate) строго меньше **0.05**.
- **Гибкий поиск**: Сопоставляет строки по идентификаторам, проверяя одновременно столбцы `target_id` и `locus` в таблицах DEG на соответствие `Locus Tag` в таблице TF.
- **Статистика**: Выводит в консоль краткий отчет о количестве успешно сопоставленных данных.

## Требования

Для работы скрипта необходим Python версии 3.6 или выше, а также установленные библиотеки `pandas` и `openpyxl`.

Установка зависимостей:
```bash
pip install pandas openpyxl
```

## Структура входных данных

Для корректной работы таблицы должны иметь следующую структуру:

1.  **Таблица TF**: Должна содержать столбец `Locus Tag`. Остальные столбцы сохраняются в неизменном виде.
2.  **Таблицы DEG (1 и 2)**: Должны содержать столбцы:
    - `target_id` или `locus` (используются для сопоставления);
    - `logFC` (переносится в итоговую таблицу);
    - `FDR` (используется для фильтрации < 0.05).

## Использование

Скрипт запускается через терминал (командную строку). Необходимо передать пути к четырем файлам в строгом порядке:

```bash
python DEGTF.py <TF_файл> <DEG1_файл> <DEG2_файл> <Выходной_файл>
```

### Пример команды:
Если в названиях файлов есть пробелы, обязательно используйте кавычки:
```bash
python DEGTF.py "TF_data.xlsx" "DEG_24h.xlsx" "DEG_72h.xlsx" "Result_Summary.xlsx"
```

## Выходные данные

Скрипт создает копию таблицы TF и добавляет в конец четыре новых столбца:
- `24h`: Значения `logFC` из первой таблицы DEG.
- `FDR24`: Значения `FDR` из первой таблицы DEG.
- `72h`: Значения `logFC` из второй таблицы DEG.
- `FDR72`: Значения `FDR` из второй таблицы DEG.

Если данные не прошли фильтр по FDR или идентификатор не был найден, ячейки останутся пустыми.

## Логика обработки (Data Flow)
1. Загрузка основной таблицы транскрипционных факторов.
2. Проверка таблиц DEG на наличие значимых генов (FDR < 0.05).
3. Маппинг (сопоставление) через словари для обеспечения высокой скорости работы.
4. Сохранение итогового файла в формате `.xlsx`.
